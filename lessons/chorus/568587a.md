# Lesson — Any tool that writes state in its CWD will eventually pollute your repo

## What happened
Each role-session claude REPL writes a local `.claude/` directory inside its own cwd. Discovered during a soak when `domains/hello-chorus/.claude/` appeared in `git status` after the first /loop tick. Fix: add `**/.claude/` to `.gitignore` so it can never accidentally land in a commit.

## Transferable lesson
Tools that store config or session state in their working directory (claude, node, terraform, .venv, .DS_Store, IDE caches) will drop those dirs into whatever cwd they run from. If your project runs such a tool from a non-root cwd — especially under automation — preemptively gitignore the tool's known state directory globally (`**/.tool-state/`) before it appears, not after a near-miss commit.

## Surface area
Next time you launch a third-party tool from a project subdirectory (not the repo root), check its docs for "what files do you write in CWD" and add them to .gitignore in the same commit.
