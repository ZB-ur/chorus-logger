# Lesson — Static review can't catch external-shape mismatches; every external boundary needs a runtime shape assertion or recorded fixture

## What happened
quota-watcher returned HTTP 401 on every fire because the macOS keychain entry for "Claude Code-credentials" stores a JSON wrapper (`{"claudeAiOauth": {"accessToken": ..., "refreshToken": ..., ...}}`), not a raw bearer token. The previous get_token() passed the whole blob as the Bearer header. Static review (cycle-4-pre) had checked code internal consistency and threshold logic and missed this entirely; the bug only surfaced at T1 boot. Fix detects JSON, extracts accessToken, keeps a raw-string fallback. An 8-case smoke with mocked subprocess covered all paths.

## Transferable lesson
The biggest lies in your system live at the boundary where data enters. Code review can prove a function self-consistent but cannot prove its input has the shape the function assumed. Two affordable mitigations: capture a one-time shape probe as a committed fixture, OR add a runtime shape assertion that fail-fasts with a clear message. Pick one for every external input source.

## Surface area
Next time you read from a keychain, env var, vendor API, or any external store, add a `assert_shape(data)` call (or commit a sample fixture) in the same PR — don't trust the docs alone.
