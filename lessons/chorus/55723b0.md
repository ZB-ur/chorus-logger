# Lesson — Centralize every I/O call against the swappable substrate in one file

## What happened
All inbox read/write goes through `lib/medium.sh`; the global pause check goes through `lib/gate-check.sh`. Header comments name them as the *sole* surface for that concern. Substrate today = filesystem markdown; spec says swapping to GitHub Issues later means changing only medium.sh.

## Transferable lesson
When you suspect the storage substrate will change but don't want to pay for an adapter pattern up front, gate every read/write through a single shell function file with a documented "this is the only place that touches it" comment. You get the swap-ability of an interface without the ceremony, and a single file is easier to audit than a folder of adapters.

## Surface area
Next time you reach for an abstract adapter interface "just in case the backend changes," write one `lib/<substrate>.sh` with a header that forbids direct access instead.
