# Lesson — Fix a single root cause at three layers: process rule, runtime defense, better mechanism

## What happened
The quota-watcher 401 (F7, keychain JSON shape mismatch) was traced to a deeper "external boundaries aren't shape-checked" problem. The v0.5 spec addresses it at three layers in one commit: #11 — CONVENTIONS addendum requiring visibly-synthetic test fixtures (a process rule); #9 — inline shape assertions inside the quota-watcher (a runtime defense); #7 — `lib/inject-loop.sh` two-stage probe-and-inject replacing the dead `/loop` send block (a better mechanism altogether). Spec landed under-budget at 479/600 lines after five reviewer cycles.

## Transferable lesson
When a bug exposes a class of failures, ask "what would have prevented this at design time, what would catch it at run time, what would have made the failing mechanism unnecessary?" Often the right fix is all three. A purely defensive fix leaves the rotten mechanism in place; a purely mechanical fix doesn't generalize.

## Surface area
Next time you trace a single bug to a category (boundaries, timing, idempotency, etc.), draft three patches — convention/process, runtime check, mechanism replacement — and decide which subset is worth shipping.
