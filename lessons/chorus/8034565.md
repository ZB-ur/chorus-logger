# Lesson — Test fixtures for secret-shaped data must use visibly-synthetic prefixes

## What happened
CONVENTIONS.md added a rule: test fixtures for security-sensitive types (API keys, tokens, credentials) must use prefixes that do not collide with real values. Driven by a cycle-4 incident where the executor's `sk-extracted-xyz` / `sk-raw-legacy` fixtures deliberately avoided the real `sk-ant-` prefix, which made grep-based leak detection trivially reliable. No CI hook, no automated check — adopted as a text rule enforced by review.

## Transferable lesson
Use real-shaped but visibly-fake values in tests: same format, distinct prefix or marker (`sk-test-`, `fake-`, `xxxxx`). This gives grep-for-leaks a deterministic signal — every match in committed code is either a real leak or a convention violation, never ambiguous. Don't always reach for a CI hook; sometimes a text rule + review is enough and avoids the maintenance cost.

## Surface area
Next time you write a test against a credential/token/key API, pick a prefix that is structurally valid but obviously not real, and document the convention even if it's just one line in CONVENTIONS.md.
