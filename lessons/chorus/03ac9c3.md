# Lesson — Fail loudly when a parser returns nothing; narrow bare-except to specific exceptions

## What happened
First live boot: PyYAML wasn't installed, so the regex fallback ran — but a buggy split-then-prepend dropped role names, parser produced empty rows, and the script silently exited 0 with no tmux session. Fix: (a) narrow `except Exception` to `except ModuleNotFoundError` so real parse errors surface, (b) add explicit `parsed 0 roles → exit 3` and `no PIDs in pid-file → exit 4` aborts.

## Transferable lesson
Two compounding bugs hid each other: a too-broad except swallowed parser errors, and a zero-output result was treated as success. Two rules: catch the specific exception you expect (ModuleNotFoundError, not Exception), and never let "parsed 0 of N" return success — emptiness from a parser is almost always a bug.

## Surface area
Next time you write a fallback parser, write the "if result is empty, abort with non-zero" line *before* the parser body, and replace any `except Exception` with the narrowest type that justifies the fallback.
