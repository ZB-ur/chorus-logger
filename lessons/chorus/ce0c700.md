# Lesson — Use the vendor's official telemetry endpoint, don't re-derive it from local logs

## What happened
Built a quota guard that reads the OAuth token from macOS keychain, calls the official `api.anthropic.com/api/oauth/usage` endpoint, and writes a pause-flag when 5h ≥80% or 7d ≥90%. An earlier candidate would have parsed local JSONL session files to accumulate token counts; that path was rejected once the official endpoint was discovered.

## Transferable lesson
Before reverse-engineering a vendor's accounting from local artifacts, scan their public API for an authoritative endpoint. Local re-derivation drifts (rate limit windows, refund logic, multi-device usage) and locks you to one client version. A 30-line HTTP call against the source of truth beats a parser that breaks every release.

## Surface area
Next time you find yourself summing/parsing local logs to estimate vendor-side state (quota, billing, rate limits), search the vendor's auth/usage endpoints first — there is usually one you missed.
