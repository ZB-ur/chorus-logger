# Lesson — Idle no-op should leave zero artifacts; LLM-driven prompts need a forbidden-verbs scan against the host REPL's termination keywords

## What happened
Ponger's Mode B prompt handles four wake scenarios uniformly (arm catch-up, spurious mtime, real message, catch-up after downtime). Idle no-op: when inbox is empty OR `last_ts == bookmark`, the role exits silently without writing any cycle-N marker file. Crucially, a "forbidden verbs scan" reviewed every occurrence of "exit", "done", "stop", "finished", "complete" in the prompt to confirm they sit in predicate/log contexts only — not as task-termination imperatives that the claude REPL might short-circuit on.

## Transferable lesson
Two pairings worth keeping. (1) When a tick can be a no-op, write nothing at all — empty marker files (`cycle-N-noop.md`) drown the signal in noise. Detect no-op early, return silently. (2) When you author prompts driven by an LLM in a /loop, the LLM's host may treat certain verbs as termination signals. Scan your prompt for words like "exit / done / stop / complete" and confirm each one is inside a conditional or quoted context — never standing alone as the apparent instruction.

## Surface area
Next time you write a periodic-tick agent prompt, grep your prompt for termination-adjacent verbs and verify each one lives inside a predicate; also make idle-tick paths write nothing.
