# Lesson — Stylized smoke mocks test your hypothesis of the real surface, not the real surface itself

## What happened
Task 11 first reboot: both roles timed out at Stage 1 in 30s. Real claude REPL v2.1.146 renders UI chrome (status bar, context bar, mode hint, blank padding) BELOW the `❯` prompt line, so the "last non-blank line ends with ❯" heuristic — verified against a smoke mock that emitted `❯` at literal last line — never triggered. Same shape-mismatch class as the earlier F7 keychain JSON bug. BLOCKED log was committed BEFORE the fix landed, mirroring the v0 `BLOCKED-task-9.md` precedent.

## Transferable lesson
A smoke mock simulates your hypothesis of the real surface. If you didn't capture the real surface once and shape the mock from it, your test certifies the hypothesis, not the behavior. Probe the actual external surface (curl the real API, capture-pane on the real TUI, dump the real keychain entry) and freeze that as a fixture before writing assertions. Also: commit the BLOCKED diagnostic before the fix — it's the evidence chain that the next maintainer trusts.

## Surface area
Next time you write a smoke mock for an external surface (TUI, API, file format), capture one real sample first and shape the mock from it; if you can't, mark the test "hypothesis-based" so a future failure is interpretable.
