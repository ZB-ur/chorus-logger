# Lesson — Cap shape-assertion budgets per script and emit OS-specific deferrals with a clear message

## What happened
monitor-inbox.sh gained exactly two shape assertions, exhausting a per-script budget of ≤2 (set by a prior cycle's "assertion bloat" finding). A2: inbox path must be a regular file at startup. A1: BSD `stat -f %m` must return an integer at first read; if not, exit 1 with an explicit "Linux deferral" message rather than parsing garbage silently. A `last-emit` file (printf-written ISO-8601, NOT touch -t) supports out-of-band supervision.

## Transferable lesson
Defensive checks accumulate without discipline. A stated budget ("≤N assertion sites per script") forces ranking: which two checks would catch the most cases? Also: when a shell command's output differs across OS (BSD vs GNU `stat`, `date`, `sed -i`), emit a one-line "this branch is deferred to platform X" message instead of falling back silently. Silent fallbacks become the next BLOCKED log.

## Surface area
Next time you reach for "one more assertion just in case," check the budget; if over, replace a weaker one. Next time your shell script runs on multiple OS families, branch explicitly and emit a deferral message rather than silently using whichever flavor's syntax.
