# Lesson — Critical guards run from the OS scheduler, not from the runtime they guard

## What happened
The quota-watcher is wired via a launchd plist (5-minute `StartInterval`, `RunAtLoad`, stdout/stderr to the runtime's log dir). It is deliberately NOT scheduled by the chorus runtime itself, and `launchctl load` is left as a manual one-time bootstrap step rather than baked into `chorus-start.sh`.

## Transferable lesson
A kill-switch must outlive the thing it switches off. If the runtime owns its own pause mechanism, a runtime crash takes the brake with it. Push lifecycle-independent guards to the OS layer (launchd, systemd, cron) and accept the one-time manual install as the price of independence. Don't auto-install OS-level units from app scripts — that couples them again.

## Surface area
Next time you are about to schedule a watchdog from inside the process it watches, move it to launchd/systemd and document the install step instead.
