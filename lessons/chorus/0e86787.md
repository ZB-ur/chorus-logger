# Lesson — When a later finding overrides an earlier one, annotate — don't rewrite

## What happened
Cycle-1 reviewer applied four refinements to the retrospective. F3's "12s is empirically sufficient" claim was contradicted by F6's reproduction, but the original F3 text was kept intact with a "Superseded by F6" note inline; the v0 tag was deliberately NOT moved. A separate v0-build tag was created for the pre-F6 snapshot so both refs stay available.

## Transferable lesson
Two competing instincts to resist after a finding is overturned: (a) rewriting the original to match the new truth — destroys the reasoning chain that future-you needs to re-derive trust, (b) moving the release tag to a "better" commit — breaks every link that references the old SHA. Annotate-in-place plus a new tag preserves both history and reproducibility.

## Surface area
Next time a follow-up review invalidates an earlier finding, write a "Superseded by X" note inside the original, leave the original prose alone, and add a new tag rather than moving the old one.
