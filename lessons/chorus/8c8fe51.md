# Lesson — When the runtime is ready for real workload, disable the toy domain in the same commit — don't let toy noise drown the dogfood signal

## What happened
A 4-role `compounding` domain (executor, reviewer, summarizer, conductor-listener) was added as the first real dogfood of the chorus runtime — and the previous `hello-chorus` toy domain was explicitly disabled in `roles.yaml` in the same commit. ~547 lines added across role-prompts, README, bootstrap script, and roles.yaml edits.

## Transferable lesson
Toy domains/fixtures/demos exist to validate plumbing; once a real workload starts running on the same plumbing, every toy tick is noise that drowns the signal you actually want to read. Disable the toy in the same commit as the first real workload — don't leave both enabled "for now." If you ever need the toy back (regression check), it's still in git history with a one-line config change to re-enable.

## Surface area
Next time you ship a system from "validating mechanism" to "first real workload," include the toy/demo disable line in the same commit so the runtime cleanly transitions from PoC to production traffic.
