# Lesson — When scripting against an interactive TUI, the welcome banner can swallow input

## What happened
The boot script sent `claude --bypassPermissions` then slept 5s before piping in `/loop ...` via tmux send-keys. Discovered: claude's welcome banner was still painting at the 5s mark, and the repaint swallowed the Enter — `/loop` sat in the input buffer, never registering CronCreate. Bump to 12s fixed it empirically.

## Transferable lesson
Interactive TUIs (REPLs, shells, claude, mosh, etc.) often repaint asynchronously for several seconds after the process is "ready." A repaint that occurs while you're sending keystrokes can eat them. When automating against a TUI, either (a) wait for a stable signal (specific prompt text in the pane) rather than a sleep, or (b) over-sleep generously — the cost of an extra few seconds at boot is trivial vs. a flaky automation.

## Surface area
Next time you tmux-send-keys, expect to be paying for clock-based timing later — prefer "wait until pane contents match X" if the TUI gives you a stable string to match.
