# Lesson — Use the file that signals semantic state as your sentinel — directories can outlive the data they once held

## What happened
The `init` guard was originally `if (fs.existsSync(bonfireDir))` — but after `archive` runs, the parent `.bonfire/` directory still exists (it contains the `archive/` subdir holding archived cases). Re-initializing failed even though no active case was in progress. The fix: check `fs.existsSync(path.join(bonfireDir, 'state.json'))` — `state.json` only exists when there's an active case, regardless of what subdirs sit underneath.

## Transferable lesson
Directory-existence is a weak sentinel — directories survive partial cleanups, hold archived data, can be touched by other tools. File-existence is stronger when that file is the actual semantic marker of the state you care about. For "is there an active case?" check the case-state-file; for "is the workspace initialized?" check a manifest, not the parent dir. Same trap with "is this Python project?" — `pyproject.toml` (semantic) beats `src/` (structural).

## Surface area
Next time you write a guard like `if directory exists`, ask whether the directory survives the state-change you're trying to detect. If yes, switch to checking the file that genuinely tracks the state.
