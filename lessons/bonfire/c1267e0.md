# Lesson — Cross-link rendered notes with `[[wiki-style]]` pointers; pick a syntax your target reader (Obsidian, Foam, Logseq) already understands

## What happened
Every stage template includes inline navigation: `← [[30-c-requirements]] | → [[50-e-closure]]`. The double-bracket Obsidian-style link is the native cross-reference syntax for the target reader (Obsidian, which the bundle is rendered for per `obsidian-layout.md`). Stage D's body iterates `challenges` / `proposals` / `follow_up_questions` matching the d-critique delta schema's field names.

## Transferable lesson
Rendered output meant to be navigated as a graph (not just a sequence) needs cross-links between notes. Pick the syntax the target reader understands — `[[X]]` for Obsidian/Foam/Logseq, `[X](X.md)` for plain Markdown viewers, `<X>` for Roam. Putting prev/next links at the top of each note ("← prev | → next") makes the bundle navigable without an index. Same idea for runbooks, design docs, RFC bundles — the cross-link is the difference between a folder of files and a knowledge graph.

## Surface area
Next time you generate a bundle of related Markdown notes, decide which reader they're targeting and use that reader's cross-link syntax; put prev/next links at the top of each note.
