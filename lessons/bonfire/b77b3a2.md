# Lesson — When the last stub is replaced, delete the `stub` function itself; gate mutating commands by a schema-declared allowlist

## What happened
All remaining stub handlers were wired to real implementations — and the `stub()` function itself was deleted from the dispatcher ("no remaining references"). `preflight-update` enforces `schema.preflight_mutable_fields`: any field NOT in the allowlist is rejected, regardless of caller intent. `hooks/bonfire-dual-write.js` watches specific `.bonfire/` paths via regex and triggers re-render only on the listed patterns.

## Transferable lesson
Two paired habits. (1) When the last consumer of a placeholder/scaffold disappears, delete the placeholder — leftover `stub()` or `TODO()` functions read as available APIs and confuse the next maintainer. The compiler/linter will flag any missed call site immediately. (2) For commands that mutate state, declare the mutable-field allowlist in the schema, not the code; the operator (or attacker, or careless caller) can't update a field unless it's on the list. Same principle for PATCH endpoints, config-set CLIs, admin APIs.

## Surface area
Next time you replace the last stub in a dispatcher, grep for the stub function name and delete it. Next time you write a "set arbitrary field" command, gate it on a schema-declared allowlist before touching the data.
