# Lesson — Event-source the state ledger: append to JSONL history, regenerate snapshot via replay; never mutate the snapshot directly

## What happened
The truth surface is a dual-layer constraint ledger: a `constraint-ledger-history.jsonl` (append-only event log) is the source of truth; `constraint-ledger-snapshot.json` is a regenerated cache produced by `replay(root)` reading the history and folding events into entries. Operations (propose/update/freeze/supersede/discard) emit events to history; snapshot rewrites are atomic. Maturity gates per category (challenged_by_non_empty, evidence_required, refs_valid) live in the schema, not the code.

## Transferable lesson
For any state that needs an audit trail (constraints, decisions, approvals, edits): append-only event log = source of truth, snapshot = cache regenerated by replay. Never mutate the snapshot directly — every change appends an event, then replay produces the new snapshot. Costs: O(history-length) replay per change. Benefits: complete audit trail, time travel, no in-place corruption, easy debugging. Drive policy (maturity gates, allowed transitions) from a schema so changes don't require code edits.

## Surface area
Next time a system needs "see who changed what when" plus current-state queries, use append-only events + regenerated snapshot from replay; resist the urge to update the snapshot in-place "for performance."
