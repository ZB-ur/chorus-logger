# Lesson — When both a spec checklist and a template render the same handoff, the template's section headers must match the spec's enumerated items 1:1

## What happened
`templates/code-handoff.md` has section headers that mirror `handoff-quality-bar.md`'s enumerated freeze surface exactly: `Retained Goal`, `Implementation Scope`, `Frozen Product Decisions`, `Implementation Units`, `Verification Commands`, `Acceptance Checks`, `Reentry Triggers`. Each implementation unit also renders with `id/title/objective/scope/depends_on/done_when` — matching the spec's per-unit checklist. A reader can grep the same name in spec, schema, template, and JSON source.

## Transferable lesson
When you have a spec ("the handoff must include X, Y, Z") and a template ("render the handoff like this"), the section names should be identical and ordered the same way. Mismatch creates drift bugs: spec says `frozen_product_decisions` but template renders "Product Constraints," and a future maintainer "fixes" the template without realizing it's the spec's contract surface. Tools can grep both files for the same field name and verify alignment. Header names are the API between spec and template.

## Surface area
Next time you write both a spec checklist and a rendering template for the same artifact, name the template's section headers identically to the spec's enumerated items — and prefer the same order so visual diff stays cheap.
