# Lesson — When real-world results miss the spec threshold, preserve the threshold verbatim and append an "Empirical Finding" section — never weaken the bar to match reality

## What happened
v0 smoke returned 47.6% against the SPEC's ≥80% Done Condition. Rather than rewrite the threshold, the SPEC.md got an appended `## v0 Empirical Finding (2026-05-18)` section preserving ≥80% verbatim and stating the real result + what it reveals. The retrospective separately investigated WHY Task 12's two-stage review process didn't run on the failing task — turning a process gap into the cc-workflow v1 input list.

## Transferable lesson
A spec is a contract about what "good" means. The temptation when reality misses is to relax the contract; the cost is losing the comparison frame for next time. Instead, preserve the original threshold and append the empirical reality as evidence — the gap is the input to the next iteration's design. Also: when a process gate failed to fire on the bad commit, the postmortem must investigate WHY the gate was bypassed, not just what the bad code did.

## Surface area
Next time real results miss a spec threshold, do not modify the threshold — append an "Empirical Finding" section preserving the original number, and add a "why did the gate not fire?" subsection if the missed result also bypassed a quality gate.
