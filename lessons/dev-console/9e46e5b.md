# Lesson — Write a None-safe combinator for "max/min/sum of optionals", and document any signal you deliberately don't use inline

## What happened
`_build_project` composes the field-extractors into a `Project`. Two design touches: (1) a `_max_optional_date(a, b)` helper because Python's built-in `max(a, b)` raises on `None`; (2) `mtime_date` is computed ONLY when commit_date is None — with an inline comment: "Use mtime only when there is no git history; commit date is authoritative for git projects so that old repositories are not masked by recent clones."

## Transferable lesson
When you compose pure extractors that each return `Optional[T]`, you need explicit combinators — `max`, `min`, `sum`, `+` all behave wrong with None in Python. Write the four-line helper that returns the present value when one is None. Also: when a signal seems combinable but you deliberately drop it for a reason, the inline comment naming the reason is what stops the next maintainer from "fixing" it back into the bug.

## Surface area
Next time you orchestrate optional-returning functions, write `_max_optional`/`_min_optional`/`_first_present` helpers and use them; and when you intentionally drop one signal, write a one-line `# reason: ...` next to the omission.
